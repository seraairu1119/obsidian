# 面接用自己分析

# Interview Self Analysis

## Core Rule

Use this skill to keep interview answers consistent with the user's current self-analysis. For ES-style written answers, also use `es-writing-rules`. For factual profile details, also use `personal-es-profile`.

Do not invent facts, awards, numbers, or motivations. If a question requires an unknown detail, either omit it or ask briefly.

## User's Self-Analysis Core

The user's central identity:

> チームで課題に向き合い、意見を出し合いながら改善を積み重ね、成果につなげることにやりがいを感じる人。

Use the following as the stable backbone.

- 価値観: 仲間と同じ目標に向かって試行錯誤し、努力や改善の積み重ねが成果として表れることにやりがいを感じる。
- 強み: 分からないことや複雑な課題に対して、情報を調べ、理由や根拠を整理し、周囲の意見も踏まえて納得感のある方向性を作れる。
- コミュニケーション: 相手の意見を否定せずに聞き、なぜそう考えたのか理由まで確認する。自分の意見も理由とセットで伝える。
- チーム観: 意見を出し合える雰囲気を重視する。単なる作業分担ではなく、各自の視点を持ち寄り改善していくチームに惹かれる。
- 弱み: より良い案を探そうとして、自分が腑に落ちるまで考えすぎ、判断に時間がかかることがある。
- 改善行動: 調べる範囲や期限を区切り、仮説段階でも理由を添えて周囲に共有し、比較検討しながら前に進める。
- 合う環境: チームで意見を出し合い、目的や理由を共有しながら改善を重ねられる環境。
- 合わない環境: 一人で完結する仕事、意見を出しにくい雰囲気、理由が共有されず指示だけで進む環境、個人競争が強い環境、改善の余地が少ない反復作業。
- スピード感: スピード重視自体は問題ない。目的や理由が共有されていれば、速く調べて提案しながら動ける。

## Episode Allocation

Avoid repeating the same episode for every question. Choose the episode by question type.

- ガクチカ / 今頑張っていること: データ解析コンペティションを主軸にする。
- ガクチカ深掘り: 書店の配本精度向上、書籍内容と地域特性の特徴量化、実課題への接続、モデル改善、説明可能性を使う。
- 異なる意見 / チームワーク / 巻き込み: アカペラの部活動を主軸にする。
- アカペラ深掘り: 熱量差、本気で大会を目指す人と両立重視の人の二分化、録音による課題共有、LINE共有、話し合いによる改善。
- 強み: 根拠を調べ、理由を含めて共有し、周囲と納得感ある判断につなげる力。長期インターンのホームページ制作を使いやすい。
- 弱み: より良い案を探しすぎて考えすぎること。長期インターンのホームページ制作で、仮説共有しながら進める改善行動を使う。
- 就活の軸 / 働きたい環境: チームで意見を出し合い、改善を積み重ねて成果につなげられる環境。
- 自己紹介 / 人物まとめ: データ解析コンペとアカペラを短く対比し、「データや対話で課題を整理し、チームで改善する人」とまとめる。

## Interview Answer Style

Answer in natural Japanese polite form. Keep oral answers concise and speakable.

Use this structure unless the user asks otherwise:

1. 結論
2. 状況・背景
3. 自分の行動
4. 結果・変化
5. 学び・仕事への接続

For follow-up questions, do not restart the whole episode. Answer the specific question directly, then add only the needed background.

Target lengths:

- アイスブレイク: 30-45 seconds.
- Standard behavioral answer: 60-90 seconds.
- Deep-dive follow-up: 30-60 seconds.
- Final self-summary: 45-60 seconds.

## Standard Phrases

Use and adapt these phrases when they fit.

- 「チームで課題に向き合い、意見を出し合いながら改善を積み重ねることにやりがいを感じます。」
- 「意見が違うときも、まず否定せずに聞き、なぜそう考えたのかを確認するようにしています。」
- 「案だけを伝えても判断しづらいと考えているため、理由や根拠も含めて共有することを意識しています。」
- 「スピード感のある環境自体は問題なく、目的や理由が共有されていれば、調べながら素早く前に進めます。」
- 「一方で、より良い案を探そうとして考えすぎることがあるため、現在は期限を決めて仮説段階でも共有するようにしています。」

## Interview Axis Draft

Use this when asked about 就活の軸 or desired environment:

> 私は、チームで意見を出し合いながら、改善を積み重ねて成果につなげられる環境で働きたいと考えています。ただ作業を分担するだけでなく、それぞれが気づいたことや改善案を出し合い、少しずつ完成度を高めていく過程にやりがいを感じるからです。スピード感のある環境でも、目的や理由が共有されていれば、自分で調べて提案しながら前に進められると考えています。

## Weakness Draft

Use this when asked about weakness:

> 私の課題は、より良い案を探そうとして、判断に時間がかかることがある点です。何かを決める際に、後からもっと良い方法が見つかるのではないかと考え、自分が腑に落ちるまで調べたり比較したりしてしまうことがあります。一方で、仕事では限られた時間の中で判断することも重要だと考えています。そのため現在は、調べる時間や判断の期限を決め、現時点で分かっている情報をもとに仮説として周囲に共有することを意識しています。

## Output Guidance

When the user sends an interviewer question, produce a ready-to-speak answer first. If helpful, add a short note with:

- 使ったエピソード
- 狙い
- 深掘りされたら補足する点

Keep the answer from sounding scripted. Avoid repeating the same sentence across multiple answers in one interview flow.
